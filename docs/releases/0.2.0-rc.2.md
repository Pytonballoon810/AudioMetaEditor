# Release Candidate: 0.2.0-rc.2

Date: 2026-03-24
Type: patch
Status: test-validated

## Scope summary

Targeted stability patch for CSP behavior in renderer audio loading.

- fixed WaveSurfer `data:` fetch blocking caused by CSP `connect-src`
- removed unsupported `frame-ancestors` directive from meta CSP
- added regression tests to prevent CSP drift

## What changed

- `index.html`
  - added `data:` to `connect-src`
  - removed `frame-ancestors` from meta CSP content
- `tests/csp.test.ts`
  - validates CSP meta presence
  - validates `connect-src` contains `data:`
  - validates `frame-ancestors` is not present in meta CSP

## Validation

Executed commands:

```bash
npm run format
npm run lint
npm run typecheck
npm run tests
```

Expected outcome:

- all quality gates pass
- WaveSurfer can load `data:audio/*` sources in development mode without CSP violations

## Notes

`frame-ancestors` must be delivered as an HTTP response header to be enforced. It is ignored in `<meta http-equiv="Content-Security-Policy">` and should remain absent from meta policy strings.
