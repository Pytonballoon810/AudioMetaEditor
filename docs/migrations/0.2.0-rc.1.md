# Migration: 0.2.0-rc.1

## What changed

- The preload bridge contract was formalized and typed via `src/ipc/contracts.ts`.
- Renderer bridge access now routes through `src/services/audioMetaApi.ts`.
- Error normalization now maps channel-prefixed IPC errors to user-facing status messages.
- Security policy blocks private/local download hosts by default.

## Impact

- Custom renderer code that accessed `window.audioMetaApi` directly should migrate to `getAudioMetaApi` / `requireAudioMetaApi`.
- Environments relying on private-network URL downloads must explicitly opt in.

## Required actions

1. Restart the desktop app after upgrade so preload changes are reloaded.
2. If using private-network download URLs in trusted environments, set:
   - `AUDIO_META_ALLOW_PRIVATE_DOWNLOADS=true`
3. Ensure bridge method names and payload types remain aligned across:
   - `electron/main.js`
   - `electron/preload.js`
   - `src/ipc/contracts.ts`
   - `src/vite-env.d.ts`

## Validation

Run:

```bash
npm run format
npm run lint
npm run typecheck
npm run tests
```

Then verify manually:

- open file and directory
- waveform load and playback
- metadata save and clip export
- URL download behavior (public host allowed, private host blocked by default)
